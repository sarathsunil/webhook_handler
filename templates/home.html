Release Notes Policy

There will be one release note per project per release

However this does not mean that the release note generated will be final, as more and more issues are added to this release, the release note will be revised to include these

When do we update Release Notes for a release

Release Notes are generated whenever there is a push to the repo which results in a successful build

Pre-requisites for autogeneration of Release Notes

The release notes will not be generated if the last commit message does not match the format JIRA ISSUE ID:comment. Example commit message: "CSIDO-10:New modules added"

The release notes will not be generated if the corresponding JIRA ITEM does not have a valid release tag in the release tag field of JIRA

The release notes will not be generated if the ISSUE-ID is already in Done state in JIRA
